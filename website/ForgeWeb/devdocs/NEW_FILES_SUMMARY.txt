================================================================================
CONFIGURATION
================================================================================

Settings stored in: forgeweb/.forgeweb_control_config.json

Example structure:
{
  "mode": "integrated",  (or "standalone")
  "services": [
    {
      "name": "ForgeWeb",
      "port": 8000,
      "health_endpoint": "http://localhost:8000/api/health",
      "enabled": true
    }
  ]
}

================================================================================
INSTALLATION MODES
================================================================================

INTEGRATED MODE:
  - Dashboard controller detected/running
  - ForgeWeb controls added to Dashboard menu
  - Single icon manages both services
  - Use: ./launch_all.sh to start both
  - Shared configuration file

STANDALONE MODE:
  - Dashboard controller not found
  - ForgeWeb runs independently
  - Own menu bar/tray icon
  - Can upgrade to integrated later
  - Independent configuration

================================================================================
WHAT GETS CREATED ON FIRST RUN
================================================================================

When you run the installer, it creates:

1. Configuration file:
   - forgeweb/.forgeweb_control_config.json

2. Integrated launcher (if Dashboard exists):
   - forgeweb/packaging/macos/launch_all.sh
   - forgeweb/packaging/linux/launch_all.sh

3. Linux desktop file (Linux only):
   - ~/.local/share/applications/forgeweb-control.desktop

4. Extracted icons (Windows):
   - Copies buildly_icon.png and forge-logo.png locally
